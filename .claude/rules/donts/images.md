# Don'ts — 이미지 / 에셋 생성

> 이미지 생성 도구 사용 시 반복된 실수. 루트 [`donts.md`](../donts.md)의 제너럴 규칙과 함께 적용.

## 🎨 API 역할 분담 (엄수)

1. **Gemini 이미지 생성은 사용자 명시 요청 시만** — 기본 이미지 파이프라인은 OpenAI (`hatchet-art.sh` → `pixel-artist`). "Gemini로 만들어줘" 명시 없이 자동 사용 금지.
2. **Gemini = 텍스트 검토/검수 (무료)**, **OpenAI = 이미지 전용 (유료)**, **Claude = 리서치/설계/구현/총괄**. 역할 섞지 말 것.

## 📋 생성 전 필수 절차

3. **이미지 생성 전 사전 승인 필수** — 바로 생성 금지. 제안서(유형/도구/크기/프롬프트/참조/경로) 제시 → 사용자 "ㅇㅇ" 받은 뒤에만 실행.
4. **예외:** 사용자가 "바로 만들어", "자동으로 해" 명시 or `hatchet-art.sh batch` 매니페스트 확정 건만 승인 생략 가능.

## 🖼️ 참조 이미지 규칙

5. **캐릭터 설정화 있으면 참조 필수** — 텍스트만으로 생성하면 매번 얼굴이 달라짐. `--ref 설정화.png` 필수.
6. **캐릭터 ref 1장만 + 도구는 텍스트로** — ref 2장(캐릭터+도구) 동시에 넘기면 얼굴 무너짐 (OpenAI edits API 한계, 2026-03-27 검증). 도구는 프롬프트 텍스트로만 묘사.

## 🖌️ 스프라이트 품질

7. **스프라이트는 2×2 그리드 방식으로 생성** — 개별 생성 금지. 4방향/4프레임을 한 번에 뽑아야 일관성 유지.
8. **생성물 품질 체크 필수** — 생성 직후 `art-reviewer`로 검수. 4대 불량 패턴: (a) 픽셀 경계에 어긋난 **직선 아티팩트**(삐뚤삐뚤한 1~2px 계단), (b) **실루엣 뭉개짐**(가장자리 대비 부족, 배경과 섞임), (c) **프레임 간 불일치**(2×2 그리드에서 팔·머리 크기 달라짐), (d) **업스케일 번짐**(blur 리사이즈). 리사이즈는 반드시 nearest-neighbor. 검수 미통과 시 재생성.

## 💾 저장 / 네이밍

9. **덮어쓰기 금지** — 같은 이름으로 재생성 금지. 버전 suffix(`_v2`, `_v3`) 또는 날짜 붙여서 히스토리 보존.

## 🔳 투명 배경 / 파일 포맷

10. **투명 배경은 `-alpha on` + `PNG32:` 필수** — ImageMagick에서 투명 처리 안 되면 이 옵션 빠진 것.
11. **favicon은 `.ico` 대신 `.png`** — 투명도·품질 안정적.

## 📐 일러스트 파이프라인

12. **설정화(SOT)는 OpenAI → 포즈/변형은 Gemini ref 1장으로** — 혼용 순서 바꾸지 말 것. SOT가 먼저 확정되어야 일관성 유지.
