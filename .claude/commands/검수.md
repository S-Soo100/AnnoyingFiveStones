# 병렬 기획 검수 (Gemini + Claude)

기획 문서의 일관성을 Gemini와 Claude 두 AI로 병렬 검수한 뒤, 결과를 종합한다.

## 사용법
- `/검수` — 기본: game-dev/specs/five-stones/ 전체 검수
- `/검수 {폴더명}` — 특정 폴더 검수

## 실행 절차

### 1단계: 대상 결정
- 인자가 있으면: `/Users/baek/ideaBank/game-dev/specs/{인자}/`
- 인자가 없으면: `/Users/baek/ideaBank/game-dev/specs/five-stones/`

### 2단계: 병렬 검수 실행
**반드시 두 에이전트를 동시에(병렬로) 실행한다:**

**에이전트 A — Gemini 검수 (백그라운드):**
```bash
cd /Users/baek/ideaBank && ./tools/consistency-check.sh game-dev/specs/five-stones/ game-dev/reports/five-stones-consistency-gemini.md
```

**에이전트 B — Claude 검수 (백그라운드):**
- subagent_type: "general-purpose", model: "opus"
- 대상 폴더의 모든 .md 파일을 읽고 아래 기준으로 검수:
  1. 수치 불일치
  2. 명칭 불일치
  3. 로직 모순
  4. 누락
  5. 미결정 사항
  6. 타임라인 충돌
- 결과를 `/Users/baek/ideaBank/game-dev/reports/five-stones-consistency-claude.md`에 저장

### 3단계: 종합 보고서 생성
두 결과를 비교·종합하여 `five-stones-consistency-final.md` 생성:
- 양쪽 공통 지적 = 확실한 이슈 (🔴)
- 한쪽만 지적 = 추가 확인 필요 (🟡)
- 개선 제안 (🟢)

### 핵심 규칙
- 검수 결과는 **반드시 파일로 저장**
- 종합 시 양쪽 공통 지적을 최우선으로 표시
